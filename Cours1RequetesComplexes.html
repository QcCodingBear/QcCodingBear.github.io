Contraintes de colonnes

La contrainte CHECK permet de valider les valeurs d’une colonne selon une condition.

Exemples :

  • CHECK (nom_colonne > 0)
  • CHECK (nom_colonne BETWEEN 1 AND 10)
  • CHECK (nom_colonne LIKE 'A%')

Utilisation avancée : dépendance entre colonnes (exemple : une date de fin doit être supérieure à une date de début).

Les Index

Les index accélèrent l’accès aux lignes d’une table et améliorent les performances des requêtes. Ils peuvent être créés automatiquement ou manuellement.

CREATE INDEX idx_nom_table ON nom_table (colonne1, colonne2);

Exemple : Créer un index sur deux colonnes :

CREATE INDEX idx_employe_nom_prenom ON employe (nom, prenom DESC);

Les Transactions

Une transaction regroupe plusieurs opérations en une unité logique de travail, garantissant la cohérence des données.

  • COMMIT: valide les modifications.
  • ROLLBACK: annule les modifications.
  • SAVEPOINT: définit un point de sauvegarde intermédiaire.

Exemple d’utilisation :

INSERT INTO table VALUES (...);
SAVEPOINT save1;
UPDATE table SET colonne = ...;
ROLLBACK TO save1;

Fonctions sur les Dates

SQL offre des fonctions pour manipuler les dates, comme SYSDATE pour la date actuelle.

Exemple : Ajouter 30 jours à une date :

SELECT SYSDATE + 30 AS "Date + 30" FROM DUAL;

Les Vues

Une vue est une table virtuelle basée sur une requête. Elle permet de simplifier les accès aux données et de limiter leur exposition.

CREATE VIEW vue_exemple AS
SELECT colonne1, colonne2
FROM table
WHERE condition;
