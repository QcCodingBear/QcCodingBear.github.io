La Gestion des Exceptions en PL/SQL

Qu'est-ce qu'une exception ?

Une exception est une erreur qui survient lors de l'exécution d'un programme PL/SQL. Il en existe deux types :

  • Exceptions implicites : Déclenchées automatiquement par le serveur Oracle (par exemple, TOO_MANY_ROWS).
  • Exceptions explicites : Déclenchées manuellement par l'utilisateur via l'instruction RAISE.

Structure d'une exception en PL/SQL

Voici comment gérer les exceptions dans un bloc PL/SQL :

DECLARE
-- Déclarations
BEGIN
-- Instructions exécutables
EXCEPTION
WHEN exception_name THEN
-- Instructions pour gérer l'exception
WHEN OTHERS THEN
-- Instructions pour toutes les exceptions non gérées explicitement
END;

Exemple : Gestion de l'exception TOO_MANY_ROWS

DECLARE
lname VARCHAR2(15);
BEGIN
SELECT last_name INTO lname
FROM employees
WHERE first_name = 'John';
DBMS_OUTPUT.PUT_LINE('Le nom de famille est : ' || lname);
EXCEPTION
WHEN TOO_MANY_ROWS THEN
DBMS_OUTPUT.PUT_LINE('Il y a plusieurs employés avec ce prénom.');
END;
/

Types d'exceptions

  • Prédéfinies : Gérées directement via leur nom (exemple : NO_DATA_FOUND, ZERO_DIVIDE).
  • Non prédéfinies : Nécessitent d’être déclarées avec PRAGMA EXCEPTION_INIT.
  • Définies par l'utilisateur : Créées et déclenchées manuellement pour des besoins spécifiques.

Exemple : Déclaration et gestion d'une exception utilisateur

DECLARE
invalid_department EXCEPTION;
BEGIN
UPDATE departments
SET department_name = 'Nouveau Nom'
WHERE department_id = 9999;
IF SQL%NOTFOUND THEN
RAISE invalid_department;
END IF;
EXCEPTION
WHEN invalid_department THEN
DBMS_OUTPUT.PUT_LINE('Aucun département avec cet ID.');
END;
/

Fonctions utiles pour les exceptions

  • SQLCODE : Retourne le code de l'erreur.
  • SQLERRM : Retourne le message associé au code d'erreur.
