Les Jointures SQL

Introduction

Les jointures SQL permettent de combiner des données provenant de deux tables ou plus en se basant sur une condition commune. Voici les principaux types de jointures et leurs utilisations.

Types de Jointures

1. Jointure Interne (INNER JOIN)

Combine les lignes des deux tables où une correspondance est trouvée entre les clés spécifiées.

SELECT <colonnes>
FROM A
INNER JOIN B
ON A.clef = B.clef;

2. Jointure Gauche (LEFT JOIN)

Renvoie toutes les lignes de la table de gauche (A), ainsi que les correspondances dans la table de droite (B). Si aucune correspondance n'est trouvée, les colonnes de B contiennent NULL.

SELECT <colonnes>
FROM A
LEFT JOIN B
ON A.clef = B.clef;

3. Jointure Droite (RIGHT JOIN)

Renvoie toutes les lignes de la table de droite (B), ainsi que les correspondances dans la table de gauche (A). Si aucune correspondance n'est trouvée, les colonnes de A contiennent NULL.

SELECT <colonnes>
FROM A
RIGHT JOIN B
ON A.clef = B.clef;

4. Jointure Complète (FULL OUTER JOIN)

Renvoie toutes les lignes des deux tables, avec des NULL pour les colonnes manquantes dans les lignes non correspondantes. Cette jointure est disponible dans certains SGBD (par exemple : Oracle).

SELECT <colonnes>
FROM A
FULL OUTER JOIN B
ON A.clef = B.clef;

Cas Particuliers

1. Lignes sans Correspondance

Pour obtenir uniquement les lignes sans correspondance dans B :

SELECT <colonnes>
FROM A
LEFT JOIN B
ON A.clef = B.clef
WHERE B.clef IS NULL;

Pour obtenir uniquement les lignes sans correspondance dans A :

SELECT <colonnes>
FROM A
RIGHT JOIN B
ON A.clef = B.clef
WHERE A.clef IS NULL;

2. Combinaison de Jointures

Pour obtenir toutes les lignes des deux tables, y compris celles sans correspondance (équivalent à FULL OUTER JOIN dans les SGBD ne le supportant pas) :

SELECT <colonnes>
FROM A
LEFT JOIN B
ON A.clef = B.clef
UNION
SELECT <colonnes>
FROM A
RIGHT JOIN B
ON A.clef = B.clef;
