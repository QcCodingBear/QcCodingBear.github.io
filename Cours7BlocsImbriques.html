Les Blocs Imbriqués en PL/SQL

Introduction

Les blocs imbriqués en PL/SQL permettent d'organiser le code en encapsulant un bloc à l'intérieur d'un autre. Ils sont particulièrement utiles pour gérer la portée des variables, les exceptions et l'allocation des ressources.

Terminologie

  • Bloc parent : Appelle et contrôle le sous-bloc.
  • Sous-bloc : Bloc imbriqué appelé par le bloc parent.

Portée des variables

La portée d'une variable est limitée au bloc dans lequel elle est déclarée. Les sous-blocs ne peuvent pas accéder aux variables du bloc parent, sauf si elles sont passées explicitement en paramètres.

-- Bloc principal
DECLARE
x NUMBER := 10;
BEGIN
-- Sous-bloc
DECLARE
y NUMBER := 20;
BEGIN
DBMS_OUTPUT.PUT_LINE('x=' || x);
DBMS_OUTPUT.PUT_LINE('y=' || y);
END;
-- Erreur si on essaie d'accéder à y ici
END;
/

Avantages des blocs imbriqués

  • Allocation différée : Les variables d'un sous-bloc ne sont créées qu'au moment où le sous-bloc est exécuté.
  • Propagation des exceptions : Les exceptions non gérées dans le sous-bloc sont transmises au bloc parent.
  • Isolation des erreurs : Les erreurs survenant dans un sous-bloc n'interrompent pas nécessairement l'exécution du bloc parent.

Gestion des exceptions

Les exceptions déclenchées dans un sous-bloc peuvent être capturées par le bloc parent, ou isolées pour éviter l'arrêt complet de l'exécution.

BEGIN
FOR emp_rec IN (SELECT * FROM emp) LOOP
BEGIN
INSERT INTO log VALUES (emp_rec.empno);
EXCEPTION
WHEN DUP_VAL_ON_INDEX THEN
DBMS_OUTPUT.PUT_LINE('Erreur pour l\'employé : ' || emp_rec.empno);
END;
END LOOP;
END;
/

Exemple complexe

Voici un exemple où un sous-bloc gère une exception spécifique et transmet l'information au bloc parent :

BEGIN
DECLARE
dept_no NUMBER := 101;
BEGIN
BEGIN
-- Sous-bloc
UPDATE departments SET dept_name = 'Nouvelle équipe' WHERE dept_id = dept_no;
IF SQL%ROWCOUNT = 0 THEN
RAISE NO_DATA_FOUND;
END IF;
EXCEPTION
WHEN NO_DATA_FOUND THEN
DBMS_OUTPUT.PUT_LINE('Aucun département trouvé pour ' || dept_no);
RAISE;
END;
END;
END;
