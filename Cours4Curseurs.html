Les Curseurs en PL/SQL

Qu'est-ce qu'un curseur ?

Chaque instruction SQL exécutée par Oracle utilise un curseur implicite, mais on peut également créer des curseurs explicites pour des besoins spécifiques.

Types de curseurs

  • Curseur implicite : Automatiquement géré par Oracle pour les instructions SQL comme INSERT, UPDATE, ou SELECT.
  • Curseur explicite : Déclaré et contrôlé manuellement par le programmeur pour traiter des jeux de données multi-lignes.

Étapes d'utilisation des curseurs explicites

  1. Déclaration : Créer une zone SQL en définissant le curseur.
  2. Ouverture : Utiliser OPEN pour exécuter la requête SQL.
  3. Extraction : Récupérer les lignes une par une avec FETCH.
  4. Fermeture : Libérer les ressources avec CLOSE.

Exemple de curseur explicite

DECLARE
  CURSOR emp_cursor IS
  SELECT employee_id, last_name FROM employees WHERE department_id = 30;
BEGIN
  OPEN emp_cursor;
  FETCH emp_cursor INTO emp_id, emp_name;
  DBMS_OUTPUT.PUT_LINE('Employé : ' || emp_name);
  CLOSE emp_cursor;
END;

Fonctionnalités avancées

Les curseurs peuvent être paramétrés, intégrés dans des boucles FOR et associés à des types composés pour plus de flexibilité dans vos programmes PL/SQL.
